ProjectTitle: AstroEditor

Краткое описание: Это IDE, включающая редактор и среду выполнения, разработанная на C# (от 10 версии) для программирования посредством вставки в строки программы заполняемых форм. Разрабатывается с взаимствованием опыта работы с Fanuc TP, Kawasaki AST, Kuka KRL, blockly.
Цель: Предоставить самодостаточную среду программирования на подобии FanucTP, с возможностями нормальных языков программирования, для обучения программированию на компьютере и/или внедрения в робототехнику (создание своих контроллеров движения).
Выполнение происходит на ИНТЕРПРЕТИРУЕМОМ языке, который представляет из себя AST структуру, которая выполняется интерпретатором.

Ключевые особенности программирования:

1) Инициализация локальных и глобальных переменных: На подобии Kawasaki AS, переменные добавляются в таблицы (реестр) своих типов данных.
Реализованы как глобальные таблицы для использования между программами, так и локальные для использования внутри программы
Таблицы сохраняются в постоянной памяти компьютера в формате JSON. Сохранение происходит при редактирвоании форм, изменении данных и фоново, как в FANUC
В коде программы отображается их значение (числовых, булевых, строковых) и в таблицах переменных можно видеть как текущее значение, так и редактирвоать его

2) Внедренная типизация. В отличии от FanucTP с его регистрами, регистрами позиций и т.д. проект позволяет использовать различные типы данных для логики. Типизация проверяется на этапе создания программы

3) Создание типов данных: Проект позволяет пользователю создавать свои типы данных в специальном редакторе типов, на основе базовых типов. Можно создавать как типы основанные на притивах с настройкой совместимости между ними и ограничениями значений, так и массивы (списки) и структуры (списки) из переменных разных типов данных. Типы данных могут создаваться в специальном редакторе, им задается категория авторства (Ядро, система, поставщик, пользователь), Имя типа данных, На каком базовом типе данных основан (примитивный тип или список/структура данных). Также Имеется функциональность псевдонимов (по сути ссылок на переменные) для смыслового переименования типов.

4) Создание программ:
Программы представляют из себя структуру, включающую в себя список строк, таблицы типов данных, типы аргументов и т.д.
Позволяет создавать программы состоящие из строк, выполнять их пошагово или без остановки.
Можно задавать группы многопоточности, комментарии, имена и типы данных аргументов, возвращаемое значение, разграничение прав редактирования и использования, вид использования (провал счетчика программы или моментальное исполнение), тип программы (процедура, функция, функция меню (например для быстрого создания строки точки для перемещения) и т.д.)
Каждая строка это атомарная операция

5) В каждую строку можно вставить выполняемую форму, состоящую из заполняемых полей. Ей можно задать категорию и имя, отображаемое в списке.
Формы можно создавать самостоятельно в редакторе форм, задавая типы полей, виды полей (Аргумент, выпадающий список, доступные ситуативные поля и т.д.) и принимаемые типы данных и выполняемую логику (программу)

6) Поля могут заполняться как выбором переменных из уже созданных переменных разрешенных полем типов данных, так и вставкой выражений, и функций. Для полей можно задавать ограничения на значение. 

7) выражения являются "мини функциями, возвращающими определенный тип данных. По сути простая вставка переменной (выбор из таблиц переменных) является выражением возвращающим ту же самую переменную.

8) Многопоточность. Пользователь может создавать как фоновые процессы, так и запускать несколько программ параллельно. Фоновые процессы нужны для обновления данных в реальном времени. Переменные можно читать всем потокам одновременно, но запись может производить только один поток/задача

9) Работа с различными протоколами и интерфейсами (TCP/IP, UDP, MODBUS TCP, PROFINET, PROFIBUS, GPIO, I2C, SPI, UART и т.д.)

10) Ввиду того что в основе лежит C#, он может выступать низкоуровневым языком программирования на подобии KAREL для FANUC TPL.
Программы могут компелироваться или выполнятсья как скриптовый язык прямо в среде.
Имется возможность редактирования кода в редакторе

11) Интерпретатор: Через инструкции можно контролировать ход выполнения программы (условия, паузы, циклы и т.д.)
Особенности среды:

1) Интерфейс написан на SharpConsoleUI.
    1.1) Это позволяет отображать его в консоли терминала, SSH, и т.п.
    1.2) Работа с несколькими окнами.
    1.3) Отображение различных меню поверх окон
    1.4) Рендер изображений
2) Имеются различные меню. Пользователь в среде может создавать их сам, для настройки и отображения данных
3) Реализовано редактирование и просмотр переменных в таблицах переменных.
4) В редактор программ можно добавить быстровызываемые функции, которые к примеру будут создавать форму перемещения к текущей позиции и локальную переменную.
5) Поддержка пользовательских расширений. Позволит внедрять функционал разработанный сообществом.
6) Пользователь может добавлять условия разрешающие работу интерпретатора ( например состояния режима выполнения, внешние факторы и т.п.). Как и других системных процессов.

... Функционал будет дополнятся основные примеры перечислены в кратком описании
